# DR0001: Use return value

## Summary

The return value of a Result-returning method should be checked to ensure failed results don't go unnoticed.

## Details

This rule flags expression statements that produce a `Result<TValue, TError>` and then ignore it. Ignoring result values can lead to silent failures and make debugging difficult.

The rule reports:

- Direct invocations that return `Result<TValue, TError>`
- Chained result operations such as `Map`, `And`, or `Or` when the final result is ignored
- Awaited expressions whose awaited value is `Result<TValue, TError>`

The rule does not report assignments, discards, returns, or unawaited `Task<Result<TValue, TError>>` values.

## Examples

### Noncompliant

```csharp
// Return value is ignored
SomeMethodThatReturnsResult();

// Chained result is ignored
SomeMethodThatReturnsResult()
    .Map(value => value * 2);

// Awaited result is ignored
await SomeAsyncMethodThatReturnsResult();
```

### Compliant

```csharp
// Assign the result to a variable
var result = SomeMethodThatReturnsResult();

// Explicitly discard the value
_ = SomeMethodThatReturnsResult();

// Chain with other operations
_ = SomeMethodThatReturnsResult()
    .Map(x => x * 2)
    .And(x => ProcessValue(x));

// Assign or discard awaited results
var asyncResult = await SomeAsyncMethodThatReturnsResult();
_ = await SomeAsyncMethodThatReturnsResult();
```

## Configuration

### Severity

```ini
# .editorconfig
dotnet_diagnostic.DR0001.severity = warning
```

### Suppression

The preferred way to suppress this rule is using the discard operator:

```csharp
// Preferred: Use discard operator
_ = SomeMethodThatReturnsResult(); // No warning
```

Alternative suppression methods:

```csharp
// Alternative: Use pragma directives
#pragma warning disable DR0001
SomeMethodThatReturnsResult(); // No warning
#pragma warning restore DR0001

// Alternative: Use attributes
[SuppressMessage("Darp.Results.Analyzers", "DR0001:UseReturnValue", 
    Justification = "Intentionally ignoring result")]
void MethodThatIntentionallyIgnoresResult()
{
    SomeMethodThatReturnsResult();
}
```

## When to Suppress

Consider suppressing this rule when:

- The result was checked by another mechanism that is not visible to the analyzer
- The result is intentionally ignored and using a discard would obscure the surrounding code
