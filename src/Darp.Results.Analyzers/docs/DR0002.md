# DR0002: Switch expression missing arm

## Summary

Switch expressions on Result types should handle both Ok and Err cases to ensure all possible states are considered.

## Details

This rule ensures that switch expressions on `Result<TValue, TError>` types handle both the success (`Result.Ok<TValue, TError>`) and error (`Result.Err<TValue, TError>`) cases.
Missing cases can lead to runtime exceptions or incomplete error handling.

This analyzer is paired with the `DR0003` suppressor, which suppresses compiler diagnostic `CS8509` for switch expressions on result types. `DR0002` provides the result-specific diagnostic and code fix instead.

## CodeFixer

This rule provides a code fixer that adds the missing `Ok` and/or `Err` switch arms with `throw new System.NotImplementedException()` placeholders.

## Examples

### Noncompliant

```csharp
// Missing Err case
var result = GetResult();
var value = result switch
{
    Result.Ok<int, string> ok => ok.Value,
    // Missing Err case
};

// Missing Ok case
var result = GetResult();
var message = result switch
{
    Result.Err<int, string> err => err.Error,
    // Missing Ok case
};
```

### Compliant

```csharp
// Handle both cases explicitly
var result = GetResult();
var value = result switch
{
    Result.Ok<int, string> ok => ok.Value,
    Result.Err<int, string> => 0, // Default value for error
};

// Use catch-all pattern
var result = GetResult();
var message = result switch
{
    Result.Ok<int, string> ok => $"Success: {ok.Value}",
    _ => "Error occurred", // Catch-all for Err case
};
```

## Configuration

### Severity

```ini
# .editorconfig
dotnet_diagnostic.DR0002.severity = error
```

## When to Suppress

Do not suppress this rule.
Not handling all cases might throw an exception which is against the intention of the result pattern.
If you have checked a specific case before, use `Expect(message)` or `ExpectError(message)` methods.
